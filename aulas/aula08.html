Razor — Misturando C# e HTML

Razor é a engine de templates que permite escrever C# dentro do HTML. Arquivos usam extensão .cshtml.

<!-- Expressão simples: @ -->
<h1>Olá, @ViewBag.Nome!</h1>
<p>Hoje é @DateTime.Now.ToString("dd/MM/yyyy")</p>

<!-- Bloco de código: @{ } -->
@{
    var mensagem = "Bem-vindo ao ASP.NET!";
    var hora = DateTime.Now.Hour;
}

<!-- Condicional em Razor -->
@if (hora < 12) {
    <p>Bom dia!</p>
} else {
    <p>Boa tarde!</p>
}

Criando um Controller

O Controller recebe requisições e decide qual View retornar. Cada método público é uma Action.

// Controllers/ProdutoController.cs
using Microsoft.AspNetCore.Mvc;

public class ProdutoController : Controller
{
    // Action: responde a /Produto/Index
    public IActionResult Index()
    {
        ViewBag.Titulo = "Lista de Produtos";
        return View();
    }

    // Action: responde a /Produto/Detalhes/5
    public IActionResult Detalhes(int id)
    {
        ViewBag.ProdutoId = id;
        return View();
    }
}

Criando uma View

A View fica em Views/[Controller]/[Action].cshtml:

<!-- Views/Produto/Index.cshtml -->
@{
    ViewData["Title"] = ViewBag.Titulo;
}

<h1>@ViewBag.Titulo</h1>
<p>Bem-vindo à página de produtos!</p>
<a href="/Produto/Detalhes/1">Ver Produto 1</a>
/Produto/Index URL no navegador
→
ProdutoController Método Index()
→
Views/Produto/Index HTML renderizado

Passando Dados para a View

ViewBag (dinâmico)

// Controller
ViewBag.Nome = "Maria";
ViewBag.Idade = 25;

// View
<p>@ViewBag.Nome</p>

ViewData (dicionário)

// Controller
ViewData["Nome"] = "Maria";

// View
<p>@ViewData["Nome"]</p>

Na próxima aula veremos a forma mais robusta: Model fortemente tipado com @model.

LinkedIn ASP.NET Rotas dinâmicas

O LinkedIn usa rotas dinâmicas para exibir perfis: /in/nome-do-usuario. Cada URL mapeia para um Controller que busca os dados do perfil e renderiza a View correspondente.
