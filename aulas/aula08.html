Padrão MVC

Model — View — Controller

O que é MVC?

O MVC é um padrão de arquitetura que separa a aplicação em três componentes principais, facilitando a organização e manutenção do código.

Model (Modelo)

Representa os dados e as regras de negócio. Define as classes que representam as entidades do sistema.

View (Visão)

Responsável pela interface com o usuário. Exibe os dados do Model usando HTML, CSS e Razor.

Controller (Controlador)

Recebe as requisições do usuário, processa a lógica e decide qual View exibir.

Fluxo de uma Requisição MVC

1. Requisição HTTP

O navegador envia uma requisição (ex: GET /Home/Index)

2. Roteamento

O ASP.NET identifica o Controller e a Action correspondentes

3. Controller

O Controller processa a lógica e obtém dados do Model

4. View

A View renderiza o HTML com os dados e retorna ao navegador

Criando o Projeto

# Criar projeto ASP.NET Core MVC
$ dotnet new mvc -n MinhaAppWeb
$ cd MinhaAppWeb

# Abrir no VS Code
$ code .

# Executar o projeto
$ dotnet run

# Ou com hot-reload (recarrega ao salvar)
$ dotnet watch run

Acesse https://localhost:5001 no navegador para ver a aplicação rodando!

Controllers e Actions

Um Controller é uma classe C# que herda de Controller. Cada método público é uma Action que responde a uma URL.

// Controllers/HomeController.cs
using Microsoft.AspNetCore.Mvc;

public class HomeController : Controller
{
    // GET: /Home/Index  ou  /
    public IActionResult Index()
    {
        ViewBag.Mensagem = "Bem-vindo ao ASP.NET Core MVC!";
        return View();
    }

    // GET: /Home/Sobre
    public IActionResult Sobre()
    {
        return View();
    }
}

Roteamento Padrão

O ASP.NET Core MVC usa o padrão de roteamento {controller}/{action}/{id?}:

URLControllerAction
/HomeControllerIndex()
/Home/IndexHomeControllerIndex()
/Home/SobreHomeControllerSobre()
/Produto/Detalhe/5ProdutoControllerDetalhe(5)

A rota padrão é configurada em Program.cs: "{controller=Home}/{action=Index}/{id?}"

Views e Razor Syntax

As Views usam a sintaxe Razor (arquivos .cshtml) que mistura HTML com código C#.

@* Views/Home/Index.cshtml *@

<h1>@ViewBag.Mensagem</h1>

<p>Data atual: @DateTime.Now.ToString("dd/MM/yyyy")</p>

@if (DateTime.Now.Hour < 12)
{
    <p>Bom dia!</p>
}
else
{
    <p>Boa tarde!</p>
}

@for (int i = 1; i <= 5; i++)
{
    <p>Item @i</p>
}

ViewBag e ViewData

São formas de passar dados do Controller para a View:

Controller

public IActionResult Index()
{
    // ViewBag (dinâmico)
    ViewBag.Nome = "Maria";
    ViewBag.Idade = 25;

    // ViewData (dicionário)
    ViewData["Curso"] = "ADS";

    return View();
}

View (.cshtml)

<h1>Olá, @ViewBag.Nome</h1>
<p>Idade: @ViewBag.Idade</p>
<p>Curso: @ViewData["Curso"]</p>

Exercício Prático 1

Criando seu Primeiro Projeto MVC

  1. Crie um projeto: dotnet new mvc -n MeuSite
  2. Execute com dotnet watch run
  3. Modifique a View Index.cshtml com uma mensagem personalizada
  4. Crie uma nova Action Sobre() no HomeController
  5. Crie a View correspondente Views/Home/Sobre.cshtml
  6. Passe seu nome e curso via ViewBag e exiba na View
  7. Adicione um link de navegação entre as páginas

Exercício Prático 2

Página de Perfil Dinâmica

  1. Crie um Controller chamado PerfilController
  2. Na Action Index(), passe via ViewBag: nome, idade, curso e lista de hobbies
  3. Na View, exiba todos os dados formatados com HTML e CSS
  4. Use @foreach para listar os hobbies
  5. Use @if para exibir "Maior de idade" ou "Menor de idade"
  6. Estilize a página usando o arquivo CSS em wwwroot/css/
