TCP/IP

O conjunto de protocolos que faz a Internet funcionar

O que é TCP/IP?

TCP/IP é um conjunto de protocolos para troca de informações em redes. Foi criado em 1971 e formalizado em 1983.

Para padronizar a criação de protocolos, foi criado o modelo OSI (Open Systems Interconnection), que define uma arquitetura de protocolos para redes. Com ele, diferentes fabricantes podem produzir equipamentos que se comunicam entre si.

Analogia: Pense no TCP/IP como o sistema postal da Internet. Assim como uma carta precisa de endereço, selo e envelope, os dados na Internet precisam de protocolos para chegar ao destino correto.

O Modelo OSI — 7 Camadas

#CamadaFunção
7AplicaçãoFunções especializadas (HTTP, FTP, SMTP)
6ApresentaçãoFormatação de dados e conversão de caracteres
5SessãoNegociação e estabelecimento de conexão
4TransporteEntrega de dados ponta-a-ponta (TCP/UDP)
3RedeRoteamento de pacotes (IP)
2EnlaceDetecção e correção de erros
1FísicaTransmissão dos bits pelo meio físico

DNS

Domain Name System — Sistema de Nomes de Domínio

O que é DNS?

O DNS (Domain Name System) converte nomes de domínio legíveis por humanos em endereços IP legíveis por máquina.

www.uninove.br

Nome que humanos entendem

192.168.1.100

Endereço que máquinas entendem

⟵ DNS ⟶

Analogia: O DNS funciona como uma agenda telefônica da Internet. Você procura pelo nome e ele retorna o número (IP).

Tipos de Domínio (TLD)

Um TLD (Top Level Domain) é o último segmento de um nome de domínio — o que aparece depois do último ponto.

TipoDescriçãoExemplos
gTLDGenéricos.com, .org, .net, .xyz
sTLDPatrocinados.gov, .edu, .mil
ccTLDCódigo de País.br, .us, .uk, .de

No Brasil: O Registro.br é a entidade responsável pelo registro de domínios .br

Estrutura de um Domínio

portal.uninove.edu.br

.br       → Primeiro nível (ccTLD)
.edu      → Domínio de Nível Superior Patrocinado
.uninove  → Segundo nível (nome da organização)
portal    → Terceiro nível (subdomínio)

Cada ponto separa um nível diferente, ajudando os computadores a encontrarem o conteúdo apropriado.

Three-Way Handshake (TCP)

O handshake de três vias é o processo usado em uma rede TCP/IP para estabelecer uma conexão entre cliente e servidor.

1. SYN

Cliente envia um pacote SYN ao servidor solicitando conexão

2. SYN-ACK

Servidor responde com SYN-ACK confirmando o recebimento

3. ACK

Cliente envia ACK final e a conexão é estabelecida

Analogia: É como um aperto de mão. "Posso conectar?" → "Sim, pode!" → "Ok, conectado!"

SSL/TLS e HTTPS

SSL (Secure Sockets Layer) é um tipo de segurança digital que permite a comunicação criptografada entre um site e um navegador. Atualmente foi substituído pelo TLS (Transport Layer Security).

HTTP (inseguro)

Dados trafegam em texto puro. Qualquer pessoa na rede pode interceptar.

HTTPS (seguro)

Dados são criptografados. Mesmo interceptados, não podem ser lidos.

Dica: Sempre verifique o cadeado na barra de endereço do navegador antes de inserir dados sensíveis!

Exercício Prático 1

Investigando o DNS

Utilizando o terminal (Prompt de Comando ou Terminal):

  1. Execute o comando nslookup www.uninove.br
  2. Anote o endereço IP retornado
  3. Tente acessar o site diretamente pelo IP no navegador
  4. Execute nslookup www.google.com e compare
  5. Execute ping www.uninove.br e observe o tempo de resposta

Exercício Prático 2

Analisando Certificados SSL/TLS

  1. Acesse https://www.uninove.br no navegador
  2. Clique no cadeado na barra de endereço
  3. Visualize as informações do certificado
  4. Anote: quem emitiu o certificado, data de validade e tipo de criptografia
  5. Repita para outros 2 sites (ex: Google, GitHub)
  6. Compare os certificados encontrados

Exercício Prático 3

Mapeando a Estrutura de Domínios

Para cada domínio abaixo, identifique os níveis (TLD, segundo nível, subdomínio):

  1. portal.uninove.edu.br
  2. www.amazon.com.br
  3. docs.google.com
  4. github.io
  5. learn.microsoft.com

Classifique cada TLD como gTLD, sTLD ou ccTLD.
