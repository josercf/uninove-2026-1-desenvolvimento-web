Classes e Propriedades em C#

Uma classe é um molde para criar objetos. As propriedades definem os dados que o objeto armazena.

public class Aluno
{
    public int Id { get; set; }
    public string Nome { get; set; }
    public string Email { get; set; }
    public int Idade { get; set; }
    public double Nota { get; set; }
}

// Criando um objeto
var aluno = new Aluno
{
    Id = 1,
    Nome = "Maria",
    Email = "maria@email.com",
    Idade = 20,
    Nota = 8.5
};

Model no MVC

No padrão MVC, o Model é a classe que representa os dados da aplicação. Ele é criado na pasta Models/.

// Models/Aluno.cs
namespace MinhaAppWeb.Models;

public class Aluno
{
    public int Id { get; set; }
    public string Nome { get; set; } = "";
    public string Email { get; set; } = "";
    public int Idade { get; set; }
    public double Nota { get; set; }
}

O Model é uma POCO (Plain Old CLR Object) — uma classe simples sem dependências de framework.

Passando Model para a View

Controller

public IActionResult Detalhe()
{
    var aluno = new Aluno
    {
        Id = 1,
        Nome = "Maria",
        Email = "maria@email.com",
        Idade = 20,
        Nota = 8.5
    };
    return View(aluno);
}

View (Detalhe.cshtml)

@model Aluno

<h2>@Model.Nome</h2>
<p>Email: @Model.Email</p>
<p>Idade: @Model.Idade</p>
<p>Nota: @Model.Nota</p>

@model define o tipo. @Model acessa os dados.

Formulários com Tag Helpers

Tag Helpers são atributos especiais do Razor que geram HTML automaticamente a partir do Model.

@model Aluno

<form asp-action="Cadastrar" method="post">
    <div>
        <label asp-for="Nome"></label>
        <input asp-for="Nome" class="form-control" />
    </div>
    <div>
        <label asp-for="Email"></label>
        <input asp-for="Email" class="form-control" />
    </div>
    <div>
        <label asp-for="Idade"></label>
        <input asp-for="Idade" class="form-control" />
    </div>
    <button type="submit">Cadastrar</button>
</form>

Recebendo Dados com [HttpPost]

public class AlunoController : Controller
{
    // GET: /Aluno/Cadastrar (exibe o formulário)
    [HttpGet]
    public IActionResult Cadastrar()
    {
        return View();
    }

    // POST: /Aluno/Cadastrar (recebe os dados)
    [HttpPost]
    public IActionResult Cadastrar(Aluno aluno)
    {
        // O ASP.NET preenche o objeto automaticamente!
        ViewBag.Mensagem = $"Aluno {aluno.Nome} cadastrado!";
        return View("Sucesso", aluno);
    }
}

O Model Binding do ASP.NET Core preenche automaticamente as propriedades do objeto a partir dos campos do formulário!

Validação com Data Annotations

using System.ComponentModel.DataAnnotations;

public class Aluno
{
    public int Id { get; set; }

    [Required(ErrorMessage = "O nome é obrigatório")]
    [StringLength(100, MinimumLength = 3)]
    public string Nome { get; set; }

    [Required(ErrorMessage = "O e-mail é obrigatório")]
    [EmailAddress(ErrorMessage = "E-mail inválido")]
    public string Email { get; set; }

    [Range(16, 100, ErrorMessage = "Idade entre 16 e 100")]
    public int Idade { get; set; }

    [Range(0, 10)]
    public double Nota { get; set; }
}

Exibindo Erros de Validação

Controller

[HttpPost]
public IActionResult Cadastrar(
    Aluno aluno)
{
    if (!ModelState.IsValid)
    {
        // Retorna o form com erros
        return View(aluno);
    }
    // Sucesso!
    return RedirectToAction(
        "Index");
}

View

<div>
  <label asp-for="Nome"></label>
  <input asp-for="Nome" />
  <span asp-validation-for="Nome"
    class="text-danger">
  </span>
</div>

<!-- Resumo de erros -->
<div asp-validation-summary="All"
  class="text-danger">
</div>

Exercício Prático 1

Formulário de Cadastro de Aluno

  1. Crie o Model Aluno com: Nome, Email, Idade, Curso e Nota
  2. Adicione Data Annotations para validação
  3. Crie o AlunoController com Actions GET e POST
  4. Crie a View com formulário usando Tag Helpers
  5. Exiba mensagens de erro de validação
  6. Ao cadastrar com sucesso, redirecione para uma página de confirmação

Exercício Prático 2

Formulário de Contato

  1. Crie o Model Contato com: Nome, Email, Assunto e Mensagem
  2. Todos os campos devem ser obrigatórios
  3. O campo Assunto deve ser um <select> com opções: Dúvida, Sugestão, Reclamação
  4. O campo Mensagem deve ter no mínimo 10 caracteres
  5. Ao enviar, exiba uma página de sucesso com os dados preenchidos
  6. Estilize o formulário com CSS (use Bootstrap se preferir)
