O que é HTML?

HTML (HyperText Markup Language) é o bloco de construção mais básico da web. Define o significado e a estrutura do conteúdo.

HTML existe desde 1991

Atualmente na versão 5, com recursos modernos para o desenvolvimento web.


Responsabilidade

Demarcar a estrutura de uma página da web.

Três pilares da Web

  • HTML → Estrutura
  • CSS → Aparência
  • JavaScript → Comportamento

Tags HTML

Os elementos HTML (tags) informam ao navegador que tipo de estrutura está sendo construída. Para que um documento seja interpretado, o arquivo deve ter a extensão .html.

<!-- Tag com fechamento -->
<p>Este é um parágrafo</p>

<!-- Tag de autofechamento (self-closing) -->
<img src="foto.jpg" alt="Uma foto" />
<br />
<hr />

Tags iniciam com <, seguidas do nome do elemento, e terminam com >. O fechamento usa /.

Estrutura Básica do HTML

<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Minha Primeira Página</title>
</head>
<body>
    <h1>Olá, Mundo!</h1>
    <p>Meu primeiro parágrafo.</p>
</body>
</html>

Principais Tags do <head>

TagFunção
<!DOCTYPE html>Informa ao navegador que é HTML5
<html>Raiz do documento, container de tudo
<head>Metadados interpretados pelo navegador
<meta>Informações sobre o documento (charset, viewport)
<title>Título exibido na aba do navegador
<link>Vincula recursos externos (CSS, favicon)
<script>Instruções JavaScript ou referência a arquivo externo

Cabeçalhos e Parágrafos

O conteúdo estruturado torna a experiência de leitura mais fácil e agradável.

<h1>Título Principal</h1>
<h2>Subtítulo</h2>
<h3>Sub-subtítulo</h3>
<h4>Nível 4</h4>
<h5>Nível 5</h5>
<h6>Nível 6</h6>

<p>Este é um parágrafo 
de texto.</p>

Existem 6 níveis de cabeçalho: <h1> a <h6>.

<h1> é o título principal e deve ser usado apenas uma vez por página.

Parágrafos são envolvidos em <p>.

Tags Semânticas

Dando significado à estrutura

O que são Tags Semânticas?

Tags semânticas possuem um significado, dando sentido à informação para o navegador e buscadores.

Benefícios

  • Melhor SEO (ranking no Google)
  • Acessibilidade para leitores de tela
  • Código mais legível e organizado
  • Facilita a manutenção

Principais Tags

  • <header> — Cabeçalho
  • <nav> — Navegação
  • <main> — Conteúdo principal
  • <section> — Seção
  • <article> — Artigo
  • <aside> — Conteúdo lateral
  • <footer> — Rodapé

Estrutura Semântica vs. Não Semântica

Sem semântica

<div class="header">
  <div class="nav">...</div>
</div>
<div class="content">
  <div class="article">...</div>
</div>
<div class="footer">...</div>

Com semântica

<header>
  <nav>...</nav>
</header>
<main>
  <article>...</article>
</main>
<footer>...</footer>

Dica: Sempre prefira tags semânticas! Use <div> e <span> apenas quando não houver uma tag semântica adequada.

Comentários em HTML

Comentários são notas no código-fonte que não são exibidas no navegador. Servem para documentar e organizar o código.

<!-- Este é um comentário em HTML -->

<!-- 
    Comentários podem ocupar
    múltiplas linhas
-->

<p>Este parágrafo será exibido</p>
<!-- <p>Este parágrafo NÃO será exibido</p> -->

Atalho no VS Code: Selecione o texto e pressione Ctrl + / para comentar/descomentar.

Exercício Prático 1

Criando sua Primeira Página HTML

  1. Crie uma pasta chamada aula03 no seu computador
  2. Abra a pasta no VS Code
  3. Crie um arquivo index.html
  4. Use o Emmet (! + Tab) para gerar a estrutura básica
  5. Adicione um título <h1> com seu nome
  6. Adicione um parágrafo <p> com uma breve apresentação
  7. Adicione subtítulos <h2> para "Formação" e "Hobbies"
  8. Adicione listas <ul> com itens em cada seção
  9. Abra com o Live Server (botão direito → Open with Live Server)

Exercício Prático 2

Página com Estrutura Semântica

Crie um arquivo semantico.html com a seguinte estrutura:

  1. Um <header> com o nome do site e um <nav> com 3 links
  2. Um <main> contendo:
    • Um <article> com título e 2 parágrafos
    • Um <aside> com informações complementares
  3. Um <footer> com informações de contato
  4. Use comentários para documentar cada seção
